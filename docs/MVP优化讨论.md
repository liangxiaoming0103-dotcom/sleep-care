## 议题2：哪些功能不好用？怎么优化？

| 功能 | 问题 | 优化建议 |
|------|------|---------|
| 医生端患者列表 | 不会自动刷新，确认授权后需手动操作 | 确认授权后自动刷新（已实现）；加 30s 定时轮询 |
| 医生端报告页 | 进入报告需先回列表再点，步骤多 | 患者卡片直接显示评分，点击即进入报告（已实现） |
| 小程序图表加载 | ECharts 首次渲染慢，显示白屏 | 加 loading 骨架屏 + `requestAnimationFrame` 延迟渲染（已实现） |
| 小程序授权流程 | 输入手机号 → 点添加 → 等待确认，步骤清晰但缺少进度感 | 授权成功后底部栏动画滑入（已实现），可加步骤引导 |
| 错误提示 | 部分错误信息是英文/技术性文字 | 统一为中文友好提示（已基本做到） |
| 注册后登录 | 注册成功返回登录页，需重新输入手机号 | 注册成功自动填充手机号（已实现） |
| 设备解绑 | 无二次确认，误触直接删除 | 加 `wx.showModal` 确认弹窗 |
| 报告页日期选择 | 默认昨天，用户可能想看今天的 | 加日期选择器，支持快速切换 |
