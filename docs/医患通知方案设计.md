# 医患通知方案设计 —— 轮询 + 红点提示

## 背景

医生在 Web 端将干预建议写入 `doctor_authorizations.doctor_note`。患者不知道、看不到。需要一个简单的"医生写 → 患者感知 → 患者查看"链路。

## 核心策略

不建新表。用客户端 `wx.getStorageSync('last_read_time')` 对比服务端 `updated_at` 来判断新内容。首页 `onShow` 轮询一个轻量检查接口。

---

## 1. 新增后端 API

只需新增 **1 个** API：

### `GET /api/patient/note/check`

**用途**：患者检查是否有医生的干预建议（供首页轮询 + 报告页展示）

**请求**：
```
GET /api/patient/note/check
Authorization: Bearer <token>
```

**响应**（有建议时）：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "has_note": true,
    "doctor_name": "张医生",
    "doctor_note": "建议每天22:30前入睡，睡前避免使用手机。近7天深睡比例偏低，建议增加日间运动量。",
    "updated_at": "2026-07-01T14:30:00.000Z"
  }
}
```

**响应**（无授权/无建议时）：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "has_note": false,
    "doctor_name": null,
    "doctor_note": null,
    "updated_at": null
  }
}
```

**后端逻辑**：
```javascript
// app.js 中新增
app.get('/api/patient/note/check', authenticateToken, async (req, res) => {
  const patientId = req.user.id;
  const db = await getDb();

  // 查询有效授权中的最新建议
  const row = dbGetOne(db, `
    SELECT a.doctor_note, a.updated_at, u.nickname AS doctor_name
    FROM doctor_authorizations a
    JOIN users u ON a.doctor_id = u.id
    WHERE a.patient_id = ?
      AND a.status = 'active'
      AND a.expire_date >= date('now')
      AND a.doctor_note IS NOT NULL
      AND a.doctor_note != ''
    ORDER BY a.updated_at DESC
    LIMIT 1
  `, [patientId]);

  if (!row) {
    return res.json({ code: 0, message: 'success', data: {
      has_note: false, doctor_name: null, doctor_note: null, updated_at: null
    }});
  }

  return res.json({ code: 0, message: 'success', data: {
    has_note: true,
    doctor_name: row.doctor_name,
    doctor_note: row.doctor_note,
    updated_at: row.updated_at
  }});
});
```

**关键点**：
- 不限制角色（患者即可调用，医生调用同样可用）
- 同时返回 `updated_at`（供客户端对比时间戳）
- JOIN `users` 返回医生姓名，前端直接展示
- 过滤掉 `doctor_note IS NULL` 或空串的记录

## 3. 红点实现

### 判断逻辑（home.js `onShow`）

```javascript
onShow() {
  // ... 现有逻辑 ...
  this.checkDoctorNote();
},

checkDoctorNote() {
  const token = app.getToken();
  wx.request({
    url: `${BASE_URL}/api/patient/note/check`,
    header: { Authorization: `Bearer ${token}` },
    success: (res) => {
      if (res.data.code === 0 && res.data.data.has_note) {
        const { updated_at } = res.data.data;
        const lastRead = wx.getStorageSync('last_note_read_time');
        // 如果从未读过，或者更新时间比上次读取时间新 → 红点
        this.setData({ hasNewNote: !lastRead || updated_at !== lastRead });
      } else {
        this.setData({ hasNewNote: false });
      }
    },
    fail: () => {} // 静默失败，不影响主页使用
  });
}
```

### 红点 UI（home.wxml）

在主页现有导航区增加一个入口按钮：

```html
<view class="nav-item" bindtap="goToReport">
  <view class="nav-icon-wrapper">
    <text class="nav-icon">💬</text>
    <view class="red-dot" wx:if="{{hasNewNote}}"></view>
  </view>
  <text class="nav-label">医生建议</text>
</view>
```

**红点消失时机**：报告页加载建议后，写入 `wx.setStorageSync('last_note_read_time', updated_at)`，下次 `onShow` 对比一致 → 红点消失。

### JS（report.js `onLoad` 或 `onShow` 中新增）

```javascript
loadDoctorNote() {
  wx.request({
    url: `${BASE_URL}/api/patient/note/check`,
    header: { Authorization: `Bearer ${token}` },
    success: (res) => {
      if (res.data.code === 0 && res.data.data.has_note) {
        const d = res.data.data;
        this.setData({
          doctorName: d.doctor_name,
          doctorNote: d.doctor_note,
          doctorNoteTime: d.updated_at ? d.updated_at.slice(0, 16).replace('T', ' ') : '',
        });
        // 标记已读
        wx.setStorageSync('last_note_read_time', d.updated_at);
      }
    }
  });
}
```

## 验证流程

1. 医生 Web 端 → 查看某患者报告 → 输入建议 → 点保存
2. 患者小程序 → 下拉首页 → `onShow` 触发检查 → 🔴 红点出现
3. 点击"医生建议"入口 → 跳到报告页 → 看到绿色左边条卡片展示建议 + "新"标签
4. 返回首页 → 红点消失
5. 医生再次修改建议 → 首页红点重新出现
