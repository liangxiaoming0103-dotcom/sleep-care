## 数据流向说明

```mermaid
sequenceDiagram
    actor U as 👤 用户
    participant MP as 微信小程序
    participant API as Express API
    participant SIM as 模拟数据生成器
    participant DB as SQLite
    actor D as 👨‍⚕️ 医生

    rect rgb(232, 245, 233)
        Note over U,DB: 🔐 用户注册/登录流程
        U->>MP: 手机号+验证码 / 微信授权
        MP->>API: POST /auth/register 或 /auth/login
        API->>DB: 写入/查询 users 表
        DB-->>API: 返回用户数据
        API-->>MP: 返回 JWT Token + 用户信息
        MP-->>U: 登录成功，进入首页
    end

    rect rgb(255, 243, 224)
        Note over U,DB: 📱 虚拟设备绑定流程
        U->>MP: 点击"添加演示设备"
        MP->>API: POST /device/add（携带 Token）
        API->>DB: 写入 devices 表（生成 VIR+16位序列号）
        DB-->>API: 返回设备信息
        API-->>MP: 返回设备列表
        MP-->>U: 展示已绑定设备
    end

    rect rgb(227, 242, 253)
        Note over SIM,DB: 🌙 睡眠数据生成（每日凌晨2点自动触发）
        SIM->>DB: 读取 users + user_settings（作息时间）
        SIM->>SIM: 按生理规则生成模拟数据<br/>（阶段曲线/心率/呼吸/噪音/事件）
        SIM->>DB: 写入 sleep_reports 表
    end

    rect rgb(243, 229, 245)
        Note over U,MP: 📊 睡眠报告查看流程
        U->>MP: 打开睡眠报告页
        MP->>API: GET /report/daily（携带 Token + 日期）
        API->>DB: 查询 sleep_reports 表
        DB-->>API: 返回报告数据（含阶段/心率/呼吸/噪音曲线）
        API-->>MP: 返回完整报告 JSON
        MP-->>U: 渲染图表（折线图/堆叠条形图）
    end

    rect rgb(252, 228, 236)
        Note over U,D: 🏥 医生授权与数据查看流程
        U->>MP: 输入医生手机号，发起授权
        MP->>API: POST /doctor/authorize
        API->>DB: 写入 doctor_authorizations 表
        API-->>MP: 授权请求已发送
        D->>WEB: 登录医生端Web
        WEB->>API: GET /doctor/patient/list
        API->>DB: 查询授权患者列表
        DB-->>API: 返回脱敏患者数据
        API-->>WEB: 返回患者列表
        WEB-->>D: 查看患者睡眠报告/日记/测评
        D->>WEB: 添加干预建议
        WEB->>API: 保存建议
        API->>DB: 写入建议记录
        MP->>API: 轮询获取医生建议
        API-->>MP: 返回医生建议
        MP-->>U: 查看医生建议
    end
```

## 技术栈说明

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 微信小程序 | 原生框架 / UniApp | 用户端交互界面 |
| 云后台 API | **Express (Node.js)** | RESTful API，JWT 认证，统一 `{code, message, data}` 响应格式 |
| 数据库 | **SQLite** | 轻量级嵌入式数据库，实训阶段免安装配置 |
| 医生端 Web | 响应式 HTML/CSS/JS | PC/Pad 适配，独立于小程序的后台页面 |
| 模拟数据生成器 | Node.js 定时任务 (node-cron) | 每日凌晨2点触发，按生理规则生成数据 |
| 未来扩展 | MQTT Broker | 预留真实硬件数据接入通道 |

---

*注：当前实训阶段不依赖真实硬件，所有设备数据、环境调控指令均使用模拟数据或预留接口。硬件就绪后，仅需替换数据源和控制指令下发模块即可无缝接入。*
