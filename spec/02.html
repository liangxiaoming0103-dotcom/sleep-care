3.2 为什么用 bcrypt 哈希密码？（约3分钟）

💡 教师口述
数据库中绝对绝对绝对不能存储明文密码！如果数据库被泄露，所有用户的密码就全暴露了。

bcrypt 是什么？
bcrypt 是一种专门用于密码哈希的算法，具有三个特点：
  • 单向性：从哈希值无法反向推导出原始密码
  • 加盐：每个密码在哈希前会加入随机"盐值"，即使两个用户密码相同，哈希结果也完全不同
  • 可调慢：可以通过调节计算轮数（cost factor）控制哈希速度，抵御暴力破解
工作流程：注册时用 bcrypt.hashSync(密码, 10) 生成哈希存入数据库；登录时用 bcrypt.compareSync(输入密码, 数据库哈希) 验证。

⚠️ 注意 bcrypt 生成的哈希值包含盐值和算法参数，长度约 60 个字符，所以 password_hash 字段要预留足够长度（TEXT 类型是安全的）。

3.3 什么是Express？（约3分钟）

💡 教师口述
Express是Node.js生态中最流行的Web框架。你可以把它理解为“帮我们处理HTTP请求的工具包”。

核心概念：
  • 路由（Route）：定义“什么路径 + 什么方法”对应“什么处理函数”。比如 app.post('/api/auth/register', ...) 就是告诉Express：当有人用POST方式访问这个路径时，执行后面的函数。
  • 中间件（Middleware）：在请求到达最终处理函数之前或之后执行的函数。比如 express.json() 是一个中间件，它把请求体中的JSON数据解析成JavaScript对象，方便我们使用 req.body。
  • 请求对象（req）：包含客户端发来的所有信息——路径、参数、请求体、请求头等。
  • 响应对象（res）：用于向客户端返回数据。比如 res.json() 返回JSON格式的响应。
打个比方：如果Node.js是“食材”，那Express就是“菜谱和厨具”——它告诉我们怎么做菜（处理请求），并提供趁手的工具（中间件、路由）。

3.4 什么是JWT？（约3分钟）

💡 教师口述
JWT（JSON Web Token）是一种用于身份认证的令牌标准。当用户登录成功后，服务器生成一个“电子通行证”（JWT令牌）发给客户端。客户端后续每次请求都带上这个通行证，服务器验证通过后就放行。

JWT的结构：由三部分组成，用点号分隔——header.payload.signature
  • Header：声明加密算法（如HS256）
  • Payload：存放用户信息（如 id、phone、role），但不能放密码等敏感信息
  • Signature：签名，用密钥对前两部分加密生成，用于验证令牌是否被篡改
为什么用JWT而不是Session？ Session需要服务器存储（占用内存），JWT是无状态的，服务器不用存。在分布式系统中，JWT更容易扩展——任何一台服务器都能验证令牌（只要知道密钥）。

🔍 代码走读：理解AI生成的数据库模块和app.js

1. db/connection.js 示例（匹配项目实际实现）

const initSqlJs = require('sql.js'); const fs = require('fs'); const path = require('path'); const DB_PATH = path.join(__dirname, '..', 'sleep_care.db'); let db = null; let initPromise = null; async function getDb() { if (db) return db; if (!initPromise) { initPromise = (async () => { const SQL = await initSqlJs(); if (fs.existsSync(DB_PATH)) { const buffer = fs.readFileSync(DB_PATH); db = new SQL.Database(buffer); } else { db = new SQL.Database(); } db.run('PRAGMA foreign_keys = ON'); return db; })(); } return initPromise; } function saveDb() { if (!db) return; const data = db.export(); const buffer = Buffer.from(data); fs.writeFileSync(DB_PATH, buffer); } module.exports = { getDb, saveDb };
  • initSqlJs()：异步加载 WASM 模块，返回 SQLite 构造器。
  • getDb() 使用 initPromise 单例模式，避免重复加载 WASM 和打开数据库文件。
  • db.run('PRAGMA foreign_keys = ON')：启用外键约束，确保数据完整性（如删除用户时级联删除其设备）。
  • saveDb()：通过 db.export() 导出二进制数据，写入磁盘文件。这是 sql.js 与 better-sqlite3 的核心区别——需要显式持久化。

2. db/schema.js 示例

const { getDb } = require('./connection'); async function initSchema() { const db = await getDb(); db.run(` CREATE TABLE IF NOT EXISTS users ( id INTEGER PRIMARY KEY AUTOINCREMENT, phone TEXT NOT NULL UNIQUE, password_hash TEXT NOT NULL, nickname TEXT NOT NULL, role TEXT NOT NULL DEFAULT 'patient', status INTEGER NOT NULL DEFAULT 1, created_at TEXT NOT NULL DEFAULT (datetime('now')), updated_at TEXT NOT NULL DEFAULT (datetime('now')) ) `); // ... 其他表（devices, sleep_reports 等） console.log('[DB] Schema initialized'); } module.exports = { initSchema };
  • 使用 CREATE TABLE IF NOT EXISTS 保证幂等性，多次运行不会报错。
  • 所有表都在这里创建，保持与第1节设计完全一致。
  • 调用 db.run() 执行 SQL 语句（sql.js 的 run 用于非查询语句）。

3. app.js 启动部分

const { initDatabase } = require('./db/init'); // ... (async () => { await initDatabase(); app.listen(PORT, () => { console.log(`🚀 服务器已启动: http://localhost:${PORT}`); }); })();

// 辅助函数：查询单行（与 sql.js 的 prepare/bind/step/getAsObject 配合使用） function dbGetOne(db, sql, params = []) { const stmt = db.prepare(sql); stmt.bind(params); let row = null; if (stmt.step()) { row = stmt.getAsObject(); } stmt.free(); return row; } // 注册接口 app.post('/api/auth/register', async (req, res) => { const { phone, password, nickname } = req.body; // 1. 校验 if (!phone || !password) { return res.json({ code:1001, message:'手机号和密码不能为空' }); } if (!/^\d{11}$/.test(phone)) { return res.json({ code:1001, message:'手机号必须是11位数字' }); } if (password.length < 6) { return res.json({ code:1001, message:'密码长度不能少于6位' }); } const db = await getDb(); // 2. 检查手机号是否已存在 const existing = dbGetOne(db, 'SELECT 1 AS _ FROM users WHERE phone = ?', [phone]); if (existing) { return res.json({ code:1001, message:'该手机号已注册' }); } // 3. bcrypt 哈希 const passwordHash = bcrypt.hashSync(password, 10); const now = new Date().toISOString(); const displayName = nickname || `用户${phone.slice(-4)}`; try { // 4. 插入用户 db.run( `INSERT INTO users (phone, password_hash, nickname, role, status, created_at, updated_at) VALUES (?, ?, ?, 'patient', 1, ?, ?)`, [phone, passwordHash, displayName, now, now] ); saveDb(); // 5. 查询新用户 const newUser = dbGetOne(db, 'SELECT * FROM users WHERE phone = ?', [phone]); return res.json({ code: 0, message: '注册成功', data: { id: newUser.id, phone: newUser.phone, nickname: newUser.nickname, role: newUser.role } }); } catch (err) { if (err.message && err.message.includes('UNIQUE constraint failed')) { return res.json({ code:1001, message:'该手机号已注册' }); } console.error('[DB] register error:', err); return res.json({ code:1001, message:'注册失败' }); } });

  • 第16-17行：使用 SELECT 1 AS _ 快速判断记录是否存在，只返回一个标记，节省数据传输。
  • 第23行：db.run() 执行 INSERT 操作（sql.js 的 run 用于非查询语句）。
  • 第30-31行：捕获 UNIQUE 约束异常，处理并发注册同一手机号的情况。
  • 关键：所有数据库操作都使用 await getDb() 确保连接已就绪，操作后调用 saveDb() 持久化到磁盘。

❗ 八、常见问题及解决方案

❌ 问题1：终端输入 claude 提示“命令未找到”

原因：Claude Code 扩展未安装或未正确配置PATH。

解决方案：

  1. 确认已在VSCode中安装 Claude Code 扩展
  2. 安装后重启VSCode，重新打开终端尝试
  3. 如果仍不行，可尝试 npx @anthropic-ai/claude-code

❌ 问题2：Claude 生成代码但未自动写入文件

原因：提示词中未明确要求“写入文件”。

解决方案：在提示词中明确加上“请直接写入文件”或手动复制粘贴。

❌ 问题3：端口3000被占用

解决方案：修改 PORT 为其他端口（如3001），或在终端关闭占用进程。

❌ 问题4：Postman 请求返回 Cannot POST

解决方案：检查URL是否正确（含端口号），方法是否为POST，Body格式是否为JSON。

❌ 问题5：登录时提示“密码错误”但密码正确

解决方案：检查密码前后是否有空格；确认注册时使用的密码；在代码中临时打印 user.password_hash 确认存储的哈希值是否与注册时一致。

❌ 问题6：数据库文件 sleep_care.db 未生成或数据未持久化

原因：未调用 saveDb()，或 saveDb() 实现有误。

解决方案：检查 db/connection.js 中 saveDb() 是否正确使用 db.export() 和 fs.writeFileSync；确保在每次写操作后调用 saveDb()。
