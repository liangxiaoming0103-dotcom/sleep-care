3.1 睡眠分期的生理意义（约3分钟）

💡 教师口述
人类睡眠是周期性的，一个完整的睡眠周期约90分钟，每晚经历4-6个周期。每个周期包含：

  • 浅睡（N1+N2）：睡眠开始阶段，约占45%-55%。此时容易被唤醒，是入睡的过渡期。
  • 深睡（N3）：恢复体力的关键阶段，约占15%-25%。此时身体修复、免疫系统增强，深睡比例是衡量睡眠质量的核心指标。
  • REM（快速眼动）：做梦阶段，约占20%-25%。此时大脑活跃，负责记忆巩固和情绪调节。
  • 清醒：夜间觉醒，正常0-3次，每次几分钟。
典型睡眠结构：入睡后先进入浅睡 → 进入深睡（前半夜深睡集中）→ 逐渐进入REM（后半夜REM集中）。这是为什么前半夜睡得“沉”，后半夜容易做梦。

3.2 分期数据的存储与读取（约3分钟）

💡 教师口述
在第4节中，我们在 sleep_reports 表中预留了 sleep_stages_json 字段（TEXT类型），用于存储分期数据。

存储格式：JSON 数组，例如 [1, 1, 2, 2, 2, 3, 3, ...]（48个数值，每10分钟一个）。

读取策略：API 先查询该字段，如果存在且非空，直接解析 JSON 返回；如果不存在或为空，则用第4节的确定性随机算法生成分期序列，更新到该字段，调用 saveDb() 持久化，然后返回。

这样，分期数据与基础指标在同一张表中，保持数据一致性，且无需额外接口。

3.3 ECharts 图表库（约2分钟）

💡 教师口述
ECharts 是百度开源的 JavaScript 图表库，功能强大、配置灵活。小程序中使用 echarts-for-weixin 组件来集成。

今天要用的图表类型：柱状图（bar）——用不同颜色的柱子表示不同的睡眠阶段，横轴是时间，纵轴是分期编码。

颜色映射：
  • 深睡（深蓝色）
  • 浅睡（浅蓝色）
  • REM（紫色）
  • 清醒（红色）
这样的配色让用户一眼就能看出“深睡集中在哪个时段”。

🔍 代码走读：睡眠分期API（数据库版）

核心逻辑（简化示例）

app.get('/api/sleep/stages', authenticateToken, async (req, res) => { const userId = req.user.id; let dateStr = req.query.date; if (!dateStr) { const yesterday = new Date(); yesterday.setDate(yesterday.getDate() - 1); dateStr = yesterday.toISOString().split('T')[0]; } const db = await getDb(); let deviceId = 0; const deviceRow = dbGetOne(db, 'SELECT id FROM devices WHERE user_id = ? LIMIT 1', [userId]); if (deviceRow) deviceId = deviceRow.id; // 1. 查询记录 let report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); // 2. 如果记录不存在，先插入基础数据（复用第4节逻辑） if (!report) { // ... 生成基础指标（sleep_score, total_sleep_minutes, etc.） db.run(`INSERT INTO sleep_reports (...) VALUES (...)`, [...]); saveDb(); report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); } // 3. 检查 sleep_stages_json 字段 let stages = []; let labels = []; if (report.sleep_stages_json) { try { stages = JSON.parse(report.sleep_stages_json); if (Array.isArray(stages) && stages.length === 48) { // 已有有效分期数据，生成标签并返回 for (let i = 0; i < 48; i++) { const hours = Math.floor(i * 0.5 / 60); const minutes = (i * 0.5) % 60; labels.push(`${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}`); } return res.json({ code: 0, message: 'success', data: { date: dateStr, stages, labels } }); } } catch (e) { /* JSON 解析失败，继续生成 */ } } // 4. 生成分期数据（确定性随机） const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const TOTAL_POINTS = 48; const newStages = []; for (let i = 0; i < TOTAL_POINTS; i++) { const position = i / TOTAL_POINTS; let deepProb = Math.max(0.1, 0.6 - position * 0.8); let remProb = Math.min(0.6, 0.1 + position * 0.5); let lightProb = 0.25 + rand() * 0.2; const r = rand(); let stage; if (r < deepProb) stage = 2; else if (r < deepProb + lightProb) stage = 1; else if (r < deepProb + lightProb + remProb) stage = 3; else stage = 0; if (rand() < 0.05 && i > 5 && i < TOTAL_POINTS - 5) stage = 0; newStages.push(stage); } // 5. 更新数据库 db.run('UPDATE sleep_reports SET sleep_stages_json = ? WHERE id = ?', [JSON.stringify(newStages), report.id]); saveDb(); // 6. 生成标签 for (let i = 0; i < TOTAL_POINTS; i++) { const hours = Math.floor(i * 0.5 / 60); const minutes = (i * 0.5) % 60; labels.push(`${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}`); } res.json({ code: 0, message: 'success', data: { date: dateStr, stages: newStages, labels } }); });
  • 先查后插：若记录不存在，先插入基础指标（复用第4节逻辑），保证数据完整性。
  • 检查 JSON 字段：若 sleep_stages_json 已有有效数据，直接返回，避免重复生成。
  • 生成分期：使用与第4节相同的 seededRandom，保证同一用户同一日期结果稳定。
  • JSON.stringify(newStages)：将数组转为 JSON 字符串存入 TEXT 字段。
  • saveDb()：更新后必须调用，否则数据只存在于内存。

💡 重点理解 这个接口实现了“懒加载”——只有在用户第一次查看某天分期数据时才会生成，之后都从数据库直接读取，节约计算资源且保证数据一致性。

为什么从 GitHub 获取而不是 npm？
官方 npm 包的设计方式与讲义中的组件引用方式不同。为了保持与讲义完全一致，我们直接从 GitHub 下载官方示例，将 ec-canvas 文件夹复制到小程序的 components 目录下。这样组件的引用路径清晰明确，学生容易理解。

  2. 复制 ec-canvas 文件夹到小程序 components 目录

    📌 Windows 用户备选 如果 cp 命令不可用，可以使用 xcopy /E /I temp-echarts\ec-canvas miniprogram\components\ec-canvas，或直接在文件管理器中手动复制粘贴文件夹。
  3. 删除临时文件夹

    npm uninstall echarts-for-weixin
🔍 为什么这样操作？（快速修复方案说明）
  • git clone：从 ECharts 官方 GitHub 仓库获取完整的示例代码，其中包含 ec-canvas 组件。
  • cp -r：递归复制整个 ec-canvas 文件夹到小程序的组件目录。
  • rm -rf：删除临时克隆的仓库，避免占用空间。
  • 为什么不用 npm 包？ 官方 npm 包 echarts-for-weixin 的设计是直接在 usingComponents 中引用包名（"ec-canvas": "echarts-for-weixin"），需要配合“构建 npm”使用。为了让组件路径更直观、便于学生理解，我们选择手动复制方式。

💡 重点理解 这种方式与第3节中“手动将需求文档复制到 docs 目录”的思路一致——直接操作文件比解释复杂的构建配置更适合实训教学。

1. ECharts 图表配置

initChart(canvas, width, height, dpr) { const chart = echarts.init(canvas, null, { width: width, height: height, devicePixelRatio: dpr }); const stages = this.data.stagesData ? this.data.stagesData.stages : []; const labels = this.data.stagesData ? this.data.stagesData.labels : []; // 定义颜色映射 const colorMap = ['#e74c3c', '#85c1e9', '#2c3e50', '#9b59b6']; // 0=清醒(红), 1=浅睡(浅蓝), 2=深睡(深蓝), 3=REM(紫) const option = { tooltip: { trigger: 'axis', formatter: function(params) { const p = params[0]; const stageNames = ['清醒', '浅睡', '深睡', 'REM']; return `${p.name}
${stageNames[p.data]}`; } }, grid: { left: '5%', right: '5%', bottom: '10%', top: '10%', containLabel: true }, xAxis: { type: 'category', data: labels, axisLabel: { rotate: 45, fontSize: 10, interval: 5 } }, yAxis: { type: 'value', min: -0.5, max: 3.5, splitLine: { show: false }, axisLabel: { formatter: function(value) { const names = ['清醒', '浅睡', '深睡', 'REM']; return names[value] || value; } } }, series: [{ type: 'bar', data: stages, barWidth: '70%', itemStyle: { color: function(params) { return colorMap[params.data] || '#ccc'; } } }] }; chart.setOption(option); return chart; }
  • colorMap：定义四个阶段对应的颜色，让图表一目了然。
  • tooltip.formatter：鼠标悬停时显示阶段名称，而非数字编码。
  • yAxis.axisLabel.formatter：将Y轴的数字编码（0-3）转换为可读的阶段名称。
  • itemStyle.color：根据数据值动态设置柱子颜色。
  • barWidth: '70%'：柱子宽度为可用空间的70%，留出间距。

2. 日期选择器

// WXML <picker mode="date" value="{{selectedDate}}" bindchange="onDateChange"> <view class="date-picker">📅 {{selectedDate}}</view> </picker> // JS onDateChange(e) { const date = e.detail.value; this.setData({ selectedDate: date }); this.loadStages(); }
  • mode="date"：日期选择模式，用户选择后返回 YYYY-MM-DD 格式。
  • bindchange：用户确认选择后触发，e.detail.value 为选中的日期。
  • this.loadStages()：日期改变后重新加载数据，图表自动刷新。

3. 页面生命周期与数据流

  • onLoad：初始化日期为昨天，调用 loadStages 加载数据。
  • loadStages：请求 API，更新 stagesData，触发图表重新渲染。
  • 数据流：用户切换日期 → onDateChange → 更新 selectedDate → 调用 loadStages → 更新 stagesData → 图表刷新。

💡 重点理解 ECharts 图表在 ec-canvas 组件中通过 initChart 函数初始化。每次数据更新后，调用 chart.setOption(option) 刷新图表——这是“数据驱动图表”的核心思想。

// ========== 睡眠分期接口（数据库版，基于 sql.js） ========== app.get('/api/sleep/stages', authenticateToken, async (req, res) => { const userId = req.user.id; let dateStr = req.query.date; if (!dateStr) { const yesterday = new Date(); yesterday.setDate(yesterday.getDate() - 1); dateStr = yesterday.toISOString().split('T')[0]; } const db = await getDb(); let deviceId = 0; const deviceRow = dbGetOne(db, 'SELECT id FROM devices WHERE user_id = ? LIMIT 1', [userId]); if (deviceRow) deviceId = deviceRow.id; // 1. 查询记录 let report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); // 2. 若不存在，生成基础指标并插入（复用第4节逻辑） if (!report) { const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const total = Math.floor(300 + rand() * 180); const deepR = 0.15 + rand() * 0.20; const remR = 0.20 + rand() * 0.05; const deep = Math.floor(total * deepR); const rem = Math.floor(total * remR); const light = total - deep - rem; const score = Math.floor(60 + rand() * 40); const awake = Math.floor(rand() * 6); const awakeMin = Math.floor(rand() * 30); try { db.run(`INSERT INTO sleep_reports (user_id, device_id, report_date, sleep_score, total_sleep_minutes, deep_sleep_minutes, light_sleep_minutes, rem_sleep_minutes, awake_minutes, awake_count, heart_rate_json, sleep_stages_json, noise_json) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`, [userId, deviceId, dateStr, score, total, deep, light, rem, awakeMin, awake, '[]', '[]', '[]'] ); saveDb(); report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); } catch (err) { // 处理重复 report = dbGetOne(db, 'SELECT * FROM sleep_reports WHERE user_id=? AND device_id=? AND report_date=?', [userId, deviceId, dateStr]); if (!report) return res.json({ code: 1001, message: '生成报告失败' }); } } // 3. 检查 sleep_stages_json let stages = []; let labels = []; if (report.sleep_stages_json) { try { const parsed = JSON.parse(report.sleep_stages_json); if (Array.isArray(parsed) && parsed.length === 48) { stages = parsed; for (let i = 0; i < 48; i++) { const hours = Math.floor(i * 0.5 / 60); const minutes = (i * 0.5) % 60; labels.push(`${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}`); } return res.json({ code: 0, message: 'success', data: { date: dateStr, stages, labels } }); } } catch (e) { /* 忽略，重新生成 */ } } // 4. 生成分期数据 const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const TOTAL_POINTS = 48; const newStages = []; for (let i = 0; i < TOTAL_POINTS; i++) { const pos = i / TOTAL_POINTS; let deepProb = Math.max(0.1, 0.6 - pos * 0.8); let remProb = Math.min(0.6, 0.1 + pos * 0.5); let lightProb = 0.25 + rand() * 0.2; const r = rand(); let stage; if (r < deepProb) stage = 2; else if (r < deepProb + lightProb) stage = 1; else if (r < deepProb + lightProb + remProb) stage = 3; else stage = 0; if (rand() < 0.05 && i > 5 && i < TOTAL_POINTS - 5) stage = 0; newStages.push(stage); } // 5. 更新数据库 db.run('UPDATE sleep_reports SET sleep_stages_json = ? WHERE id = ?', [JSON.stringify(newStages), report.id]); saveDb(); // 6. 生成标签 for (let i = 0; i < TOTAL_POINTS; i++) { const hours = Math.floor(i * 0.5 / 60); const minutes = (i * 0.5) % 60; labels.push(`${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}`); } res.json({ code: 0, message: 'success', data: { date: dateStr, stages: newStages, labels } }); });

❗ 七、常见问题及解决方案

❌ 问题1：ec-canvas 组件找不到

原因：组件文件未正确复制到 components/ec-canvas 目录，或路径配置错误。

解决方案：检查 miniprogram/components/ec-canvas/ 是否存在四个文件；检查 report.json 中的路径是否为 "../../components/ec-canvas/ec-canvas"。

❌ 问题2：图表不显示或显示空白

原因：initChart 函数未正确调用，或 echarts 核心库未引入，或数据格式不对。

解决方案：确认 report.js 顶部有 import * as echarts from 'echarts'；确认 ec-canvas 组件的 ec 属性绑定了 initChart 函数；检查 stagesData 是否有数据。

❌ 问题3：日期选择器切换后图表不刷新

原因：updateChart 未在数据更新后调用。

解决方案：确保在 loadStages 的 success 回调中调用 this.updateChart()。

❌ 问题4：柱状图颜色不正确

原因：colorMap 数组索引与分期编码不对应。

解决方案：确认 colorMap = ['#e74c3c', '#85c1e9', '#2c3e50', '#9b59b6'] 对应 0=清醒, 1=浅睡, 2=深睡, 3=REM。

❌ 问题5：Y轴显示数字0、1、2、3而不是阶段名称

原因：axisLabel.formatter 未配置或配置错误。

解决方案：在 yAxis 中添加 axisLabel: { formatter: (v) => ['清醒', '浅睡', '深睡', 'REM'][v] || v }。

❌ 问题6：echarts 引入报错 "module 'echarts' is not defined"

原因：echarts 核心库未安装或未构建 npm。

解决方案：执行 npm install echarts --save，然后在微信开发者工具中点击“工具 → 构建 npm”。或从 CDN 下载 echarts.min.js 放入 miniprogram/lib 目录，用相对路径引入。

❌ 问题7：第二次请求分期接口返回空数据

原因：sleep_stages_json 字段未正确更新或 saveDb() 未调用。

解决方案：检查 UPDATE 语句是否正确，确认 saveDb() 被调用。用 SQLite 插件查看该字段内容是否已保存。
