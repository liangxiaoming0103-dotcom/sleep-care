📚 navigateTo 与 switchTab 的区别
  • wx.navigateTo：保留当前页面，跳转到新页面（有返回按钮），适合层级跳转
  • wx.switchTab：跳转到 tabBar 页面，关闭所有非 tabBar 页面（无返回按钮），适合平级切换

🔍 为什么必须用 switchTab？
  • tabBar 页面：小程序框架会为 tabBar 页面维护独立的页面栈，switchTab 会清理其他页面栈并切换到指定 tab。

📋 四、知识讲解（教师口述 + 白板，约10分钟）

4.1 环境噪音与睡眠（约3分钟）

💡 教师口述
环境噪音是影响睡眠质量的重要因素：

  • 30-40dB：理想的睡眠环境噪音水平（相当于安静的图书馆）
  • 40-50dB：可能影响浅睡，导致觉醒次数增加
  • 50dB以上：显著干扰睡眠，影响深睡和 REM 睡眠
今天的模拟规则：
  • 夜间（22:00-06:00）：30-40dB（安静）
  • 白天（06:00-22:00）：45-65dB（日常噪音）
  • 加入随机波动，模拟真实环境
用户看到噪音曲线后，可以判断自己的睡眠环境是否理想，以及哪个时段噪音最大。

4.2 折线图与柱状图的区别（约3分钟）

💡 教师口述
上节课我们用的是柱状图（bar）展示睡眠分期，今天我们要用折线图（line）展示噪音变化：

图表类型适用场景特点
柱状图分类数据对比强调数值大小，适合离散数据
折线图趋势变化展示强调变化趋势，适合连续数据
噪音数据是连续的（每10分钟一个点），用折线图能清晰展示“夜间安静、白天嘈杂”的变化趋势。

ECharts 折线图配置：将 series.type 从 'bar' 改为 'line'，并增加 smooth: true 让曲线更平滑。

4.3 噪音数据存储与读取（约4分钟）

💡 教师口述
与分期数据类似，我们在 sleep_reports 表中预留了 noise_json 字段（TEXT类型），用于存储噪音数据。

存储格式：JSON 数组，例如 [32.5, 33.1, 34.2, ...]（144个数值，每10分钟一个）。

读取策略：API 先查询该字段，如果存在且非空，直接解析 JSON 返回；如果不存在或为空，则生成噪音序列，更新到该字段，调用 saveDb() 持久化，然后返回。

这样，噪音数据与基础指标在同一张表中，保持数据一致性。由于噪音数据与日期、用户、设备关联，使用确定性种子保证同一用户同一日期的数据稳定。

🔍 代码走读：噪音API（数据库版）

核心逻辑（简化示例）

app.get('/api/sleep/noise', authenticateToken, async (req, res) => { const userId = req.user.id; let dateStr = req.query.date; if (!dateStr) { const yesterday = new Date(); yesterday.setDate(yesterday.getDate() - 1); dateStr = yesterday.toISOString().split('T')[0]; } const db = await getDb(); let deviceId = 0; const deviceRow = dbGetOne(db, 'SELECT id FROM devices WHERE user_id = ? LIMIT 1', [userId]); if (deviceRow) deviceId = deviceRow.id; // 1. 查询记录 let report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); // 2. 如果记录不存在，先插入基础数据（复用第4节逻辑） if (!report) { // ... 生成基础指标（sleep_score, total_sleep_minutes, etc.） db.run(`INSERT INTO sleep_reports (...) VALUES (...)`, [...]); saveDb(); report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); } // 3. 检查 noise_json 字段 let noise = []; let labels = []; if (report.noise_json) { try { const parsed = JSON.parse(report.noise_json); if (Array.isArray(parsed) && parsed.length === 144) { noise = parsed; // 生成标签 for (let i = 0; i < 144; i++) { const hour = i / 6; const hours = String(Math.floor(hour)).padStart(2, '0'); const minutes = String(Math.floor((hour % 1) * 60)).padStart(2, '0'); labels.push(`${hours}:${minutes}`); } return res.json({ code: 0, message: 'success', data: { date: dateStr, noise, labels } }); } } catch (e) { /* JSON 解析失败，继续生成 */ } } // 4. 生成噪音数据（确定性随机） const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const TOTAL_POINTS = 144; const newNoise = []; for (let i = 0; i < TOTAL_POINTS; i++) { const hour = i / 6; const hourFloor = Math.floor(hour); const isNight = (hourFloor >= 22 || hourFloor < 6); let baseValue; if (isNight) { baseValue = 30 + rand() * 10; } else { baseValue = 45 + rand() * 20; } // 平滑过渡 if (hourFloor === 6) { const progress = (hour - 6) / 1; const nightValue = 30 + rand() * 10; const dayValue = 45 + rand() * 20; baseValue = nightValue + (dayValue - nightValue) * progress; } if (hourFloor === 21) { const progress = (hour - 21) / 1; const dayValue = 45 + rand() * 20; const nightValue = 30 + rand() * 10; baseValue = dayValue + (nightValue - dayValue) * progress; } newNoise.push(parseFloat(baseValue.toFixed(1))); } // 5. 更新数据库 db.run('UPDATE sleep_reports SET noise_json = ? WHERE id = ?', [JSON.stringify(newNoise), report.id]); saveDb(); // 6. 生成标签 for (let i = 0; i < TOTAL_POINTS; i++) { const hour = i / 6; const hours = String(Math.floor(hour)).padStart(2, '0'); const minutes = String(Math.floor((hour % 1) * 60)).padStart(2, '0'); labels.push(`${hours}:${minutes}`); } res.json({ code: 0, message: 'success', data: { date: dateStr, noise: newNoise, labels } }); });
  • 先查后插：若记录不存在，先插入基础指标（复用第4节逻辑），保证数据完整性。
  • 检查 JSON 字段：若 noise_json 已有有效数据，直接返回，避免重复生成。
  • 生成噪音：使用与第4节相同的 seededRandom，保证同一用户同一日期结果稳定。
  • JSON.stringify(newNoise)：将数组转为 JSON 字符串存入 TEXT 字段。
  • saveDb()：更新后必须调用，否则数据只存在于内存。
  • 平滑过渡：在6-7点和21-22点进行线性插值，避免噪音突变。

💡 重点理解 这个接口实现了“懒加载”——只有在用户第一次查看某天噪音数据时才会生成，之后都从数据库直接读取，节约计算资源且保证数据一致性。

ECharts 折线图配置

initNoiseChart(canvas, width, height, dpr) { const chart = echarts.init(canvas, null, { width: width, height: height, devicePixelRatio: dpr }); this.noiseChart = chart; const data = this.data.noiseData; if (!data) return chart; const option = { tooltip: { trigger: 'axis', formatter: (params) => { const p = params[0]; return `${p.name}
噪音: ${p.value} dB`; } }, grid: { left: '5%', right: '5%', bottom: '10%', top: '10%', containLabel: true }, xAxis: { type: 'category', data: data.labels, axisLabel: { fontSize: 10, interval: 11 // 每2小时显示一个标签 } }, yAxis: { type: 'value', min: 20, max: 80, splitLine: { show: true, lineStyle: { type: 'dashed', color: '#e8ecf0' } }, axisLabel: { fontSize: 10, formatter: (value) => value + 'dB' } }, series: [{ type: 'line', data: data.noise, smooth: true, // 平滑曲线 symbol: 'none', // 不显示数据点标记 lineStyle: { color: '#4A90D9', width: 2 }, areaStyle: { color: { type: 'linear', x: 0, y: 0, x2: 0, y2: 1, colorStops: [ { offset: 0, color: 'rgba(74, 144, 217, 0.4)' }, { offset: 1, color: 'rgba(74, 144, 217, 0.05)' } ] } } }] }; chart.setOption(option); return chart; }
  • type: 'line'：折线图类型，与柱状图的 'bar' 区分。
  • smooth: true：平滑曲线，让折线更自然。
  • symbol: 'none'：不显示数据点标记，让折线更干净。
  • areaStyle：面积填充，用渐变色从蓝色渐变到透明，增加视觉层次感。
  • interval: 11：X轴标签每2小时显示一个（144个点/24小时 = 6个点/小时，11个点间隔 ≈ 2小时）。
  • yAxis.min: 20, max: 80：固定Y轴范围，让图表对比更直观。

数据联动刷新

  • 日期切换：onDateChange 同时调用 this.loadStages() 和 this.loadNoise()。
  • 图表更新：每个数据加载成功后，调用对应的 updateChart 方法更新图表。
  • 独立实例：分期柱状图和噪音折线图使用独立的 ECharts 实例，互不干扰。

💡 重点理解 同一页面中多个 ECharts 图表的关键是 独立实例——每个 ec-canvas 组件绑定不同的 ec 对象和 initChart 函数，图表之间通过数据状态关联（共享日期），但渲染逻辑完全独立。

// ========== 噪音数据接口（数据库版，基于 sql.js） ========== app.get('/api/sleep/noise', authenticateToken, async (req, res) => { const userId = req.user.id; let dateStr = req.query.date; if (!dateStr) { const yesterday = new Date(); yesterday.setDate(yesterday.getDate() - 1); dateStr = yesterday.toISOString().split('T')[0]; } const db = await getDb(); let deviceId = 0; const deviceRow = dbGetOne(db, 'SELECT id FROM devices WHERE user_id = ? LIMIT 1', [userId]); if (deviceRow) deviceId = deviceRow.id; let report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); if (!report) { // 生成基础指标（复用第4节逻辑） const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const total = Math.floor(300 + rand() * 180); const deepR = 0.15 + rand() * 0.20; const remR = 0.20 + rand() * 0.05; const deep = Math.floor(total * deepR); const rem = Math.floor(total * remR); const light = total - deep - rem; const score = Math.floor(60 + rand() * 40); const awake = Math.floor(rand() * 6); const awakeMin = Math.floor(rand() * 30); try { db.run(`INSERT INTO sleep_reports (user_id, device_id, report_date, sleep_score, total_sleep_minutes, deep_sleep_minutes, light_sleep_minutes, rem_sleep_minutes, awake_minutes, awake_count, heart_rate_json, sleep_stages_json, noise_json) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`, [userId, deviceId, dateStr, score, total, deep, light, rem, awakeMin, awake, '[]', '[]', '[]'] ); saveDb(); report = dbGetOne( db, 'SELECT * FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); } catch (err) { report = dbGetOne(db, 'SELECT * FROM sleep_reports WHERE user_id=? AND device_id=? AND report_date=?', [userId, deviceId, dateStr]); if (!report) return res.json({ code: 1001, message: '生成报告失败' }); } } let noise = []; let labels = []; if (report.noise_json) { try { const parsed = JSON.parse(report.noise_json); if (Array.isArray(parsed) && parsed.length === 144) { noise = parsed; for (let i = 0; i < 144; i++) { const hour = i / 6; const hours = String(Math.floor(hour)).padStart(2, '0'); const minutes = String(Math.floor((hour % 1) * 60)).padStart(2, '0'); labels.push(`${hours}:${minutes}`); } return res.json({ code: 0, message: 'success', data: { date: dateStr, noise, labels } }); } } catch (e) { /* 忽略，重新生成 */ } } // 生成噪音数据 const seedKey = `${userId}_${deviceId}_${dateStr}`; const rand = seededRandom(seedKey); const TOTAL_POINTS = 144; const newNoise = []; for (let i = 0; i < TOTAL_POINTS; i++) { const hour = i / 6; const hourFloor = Math.floor(hour); const isNight = (hourFloor >= 22 || hourFloor < 6); let baseValue; if (isNight) { baseValue = 30 + rand() * 10; } else { baseValue = 45 + rand() * 20; } if (hourFloor === 6) { const progress = (hour - 6) / 1; const nightValue = 30 + rand() * 10; const dayValue = 45 + rand() * 20; baseValue = nightValue + (dayValue - nightValue) * progress; } if (hourFloor === 21) { const progress = (hour - 21) / 1; const dayValue = 45 + rand() * 20; const nightValue = 30 + rand() * 10; baseValue = dayValue + (nightValue - dayValue) * progress; } newNoise.push(parseFloat(baseValue.toFixed(1))); } db.run('UPDATE sleep_reports SET noise_json = ? WHERE id = ?', [JSON.stringify(newNoise), report.id]); saveDb(); for (let i = 0; i < TOTAL_POINTS; i++) { const hour = i / 6; const hours = String(Math.floor(hour)).padStart(2, '0'); const minutes = String(Math.floor((hour % 1) * 60)).padStart(2, '0'); labels.push(`${hours}:${minutes}`); } res.json({ code: 0, message: 'success', data: { date: dateStr, noise: newNoise, labels } }); });

❗ 八、常见问题及解决方案

❌ 问题1：登录后 tabBar 不显示

原因：登录跳转使用了 wx.navigateTo 而非 wx.switchTab。

解决方案：将 login.js 中的跳转代码改为 wx.switchTab({ url: '/pages/home/home' })。

❌ 问题2：tabBar 图标不显示

原因：图标文件不存在或路径错误。

解决方案：在 miniprogram/images 目录下创建对应的 PNG 文件，或临时在 app.json 中注释掉 iconPath 和 selectedIconPath（小程序会显示文字）。

❌ 问题3：噪音接口返回 "Cannot GET /api/sleep/noise"

原因：路由未正确添加，或 app.js 中路由顺序问题。

解决方案：检查 app.js 中是否包含 app.get('/api/sleep/noise', ...)，确保它在 app.use(express.json()) 之后定义。

❌ 问题4：噪音曲线显示为直线

原因：数据未正确加载，或 ECharts 配置中 smooth 未生效。

解决方案：在 updateNoiseChart 中检查 data.noise 是否有值；确认 smooth: true 配置在 series 中。

❌ 问题5：切换日期后噪音图表不刷新

原因：onDateChange 中未调用 loadNoise，或 updateNoiseChart 未在数据加载后执行。

解决方案：确保 onDateChange 中同时调用 this.loadStages() 和 this.loadNoise()；在 loadNoise 的 success 回调中调用 this.updateNoiseChart()。

❌ 问题6：两个图表重叠或错位

原因：WXML 中布局问题，或 ec-canvas 高度未正确设置。

解决方案：使用 flex 布局，确保每个 ec-canvas 有独立容器并设置合适高度。在 WXSS 中为每个图表容器设置固定高度。

❌ 问题7：噪音值范围不对（夜间也显示60dB）

原因：时间判断逻辑有误，isNight 条件未正确覆盖 22:00-06:00。

解决方案：检查代码中 isNight = (hourFloor >= 22 || hourFloor < 6) 是否正确。

❌ 问题8：第二次请求噪音接口返回空数据

原因：noise_json 字段未正确更新或 saveDb() 未调用。

解决方案：检查 UPDATE 语句是否正确，确认 saveDb() 被调用。用 SQLite 插件查看该字段内容是否已保存。
