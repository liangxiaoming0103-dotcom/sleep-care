3.1 日周月视图的设计思路（约4分钟）

💡 教师口述
三种视图的数据粒度不同：

视图数据粒度数据量标签格式
日视图每天一个评分最近7天"06/15"
周视图每周平均分最近6周"第25周"
月视图每月平均分最近6个月"2026-06"

核心变化（数据库版）：
  • 不再每次动态生成，而是从 sleep_reports 表读取已有的 sleep_score
  • 对于日期范围内还没有报告的日期，调用第4节的 generateReport 逻辑生成并存入数据库
  • 周/月视图在应用层对每日评分进行聚合（或用 SQL 聚合）
为什么这样做？ 因为数据一旦生成就永久保存，后续趋势分析、医生查看历史数据都基于同一份数据，结果稳定可靠。

3.2 数据聚合算法（约4分钟）

💡 教师口述
实现步骤：

  1. 获取日期范围：根据 period 和基准日期，计算需要哪些日期
  2. 逐日获取评分：对每个日期，调用 getOrCreateDailyScore(userId, deviceId, date)
    • 查询 sleep_reports 表，若有则直接返回 sleep_score
    • 若无，则生成完整报告（包含基础指标和 JSON 字段）并插入，然后返回评分
  3. 按视图聚合：
    • 日视图：直接使用每日评分
    • 周视图：按自然周分组（周日-周六），计算平均分
    • 月视图：按自然月分组，计算平均分
  4. 返回统一格式：{ labels: [...], scores: [...], avg_score: number }

边界处理：如果某天数据生成失败，使用默认评分 65 分（表示正常水平）。

3.3 ECharts markLine 参考线（约2分钟）

💡 教师口述
markLine 是 ECharts 中的“辅助线”功能，可以在图表中绘制一条水平参考线。

配置方式：
markLine: { silent: true, data: [{ yAxis: data.avg_score }], lineStyle: { color: '#e74c3c', type: 'dashed' }, label: { formatter: `平均 ${data.avg_score}分`, color: '#e74c3c', fontSize: 10 } }
  • 用户看到这条红线，就能判断自己的评分在平均水平之上还是之下
  • 这是“一眼看懂”趋势的关键设计

1. getOrCreateDailyScore 辅助函数

async function getOrCreateDailyScore(userId, deviceId, dateStr) { const db = await getDb(); // 查询是否存在 const row = dbGetOne( db, 'SELECT sleep_score FROM sleep_reports WHERE user_id = ? AND device_id = ? AND report_date = ?', [userId, deviceId, dateStr] ); if (row) return row.sleep_score; // 不存在则生成（复用第4节逻辑） const rand = seededRandom(`${userId}_${deviceId}_${dateStr}`); const total = Math.floor(300 + rand() * 180); const deepR = 0.15 + rand() * 0.20; const remR = 0.20 + rand() * 0.05; const deep = Math.floor(total * deepR); const rem = Math.floor(total * remR); const light = total - deep - rem; const score = Math.floor(60 + rand() * 40); const awake = Math.floor(rand() * 6); const awakeMin = Math.floor(rand() * 30); try { db.run(`INSERT INTO sleep_reports (user_id, device_id, report_date, sleep_score, total_sleep_minutes, deep_sleep_minutes, light_sleep_minutes, rem_sleep_minutes, awake_minutes, awake_count, heart_rate_json, sleep_stages_json, noise_json) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`, [userId, deviceId, dateStr, score, total, deep, light, rem, awakeMin, awake, '[]', '[]', '[]'] ); saveDb(); return score; } catch (err) { // 并发可能冲突，重新查询 const retry = dbGetOne(db, 'SELECT sleep_score FROM sleep_reports WHERE user_id=? AND device_id=? AND report_date=?', [userId, deviceId, dateStr]); if (retry) return retry.sleep_score; throw err; } }
  • 先查后插，保证幂等性。
  • 使用 seededRandom 保证数据确定性。
  • 插入后调用 saveDb() 持久化。
  • 异常处理中重试查询，避免并发冲突。

2. 汇总接口核心逻辑（以日视图为例）

app.get('/api/sleep/summary', authenticateToken, async (req, res) => { const userId = req.user.id; const period = req.query.period || 'day'; let baseDate = new Date(); if (req.query.date) baseDate = new Date(req.query.date); baseDate.setDate(baseDate.getDate() - 1); // 默认昨天 // 获取设备ID const db = await getDb(); let deviceId = 0; const devRow = dbGetOne(db, 'SELECT id FROM devices WHERE user_id = ? LIMIT 1', [userId]); if (devRow) deviceId = devRow.id; let labels = [], scores = []; let totalSum = 0, totalCount = 0; if (period === 'day') { for (let i = 6; i >= 0; i--) { const d = new Date(baseDate); d.setDate(d.getDate() - i); const dateStr = d.toISOString().split('T')[0]; const score = await getOrCreateDailyScore(userId, deviceId, dateStr); labels.push(`${String(d.getMonth()+1).padStart(2,'0')}/${String(d.getDate()).padStart(2,'0')}`); scores.push(score); totalSum += score; totalCount++; } } else if (period === 'week') { // 按自然周聚合 const startOfWeek = new Date(baseDate); startOfWeek.setDate(startOfWeek.getDate() - startOfWeek.getDay()); for (let w = 5; w >= 0; w--) { const weekStart = new Date(startOfWeek); weekStart.setDate(weekStart.getDate() - w * 7); const weekEnd = new Date(weekStart); weekEnd.setDate(weekEnd.getDate() + 6); let weekSum = 0, weekCount = 0; for (let d = new Date(weekStart); d <= weekEnd; d.setDate(d.getDate() + 1)) { const dateStr = d.toISOString().split('T')[0]; weekSum += await getOrCreateDailyScore(userId, deviceId, dateStr); weekCount++; } const avg = parseFloat((weekSum / weekCount).toFixed(1)); labels.push(`第${getWeekNumber(weekStart)}周`); scores.push(avg); totalSum += avg; totalCount++; } } else if (period === 'month') { // 按自然月聚合 const startOfMonth = new Date(baseDate.getFullYear(), baseDate.getMonth(), 1); for (let m = 5; m >= 0; m--) { const monthStart = new Date(startOfMonth); monthStart.setMonth(monthStart.getMonth() - m); const monthEnd = new Date(monthStart.getFullYear(), monthStart.getMonth() + 1, 0); let monthSum = 0, monthCount = 0; for (let d = new Date(monthStart); d <= monthEnd; d.setDate(d.getDate() + 1)) { const dateStr = d.toISOString().split('T')[0]; monthSum += await getOrCreateDailyScore(userId, deviceId, dateStr); monthCount++; } const avg = parseFloat((monthSum / monthCount).toFixed(1)); labels.push(`${monthStart.getFullYear()}-${String(monthStart.getMonth()+1).padStart(2,'0')}`); scores.push(avg); totalSum += avg; totalCount++; } } const avgScore = parseFloat((totalSum / totalCount).toFixed(1)); res.json({ code:0, message:'success', data: { period, labels, scores, avg_score: avgScore } }); });
  • 所有日期范围的评分都通过 getOrCreateDailyScore 获取，确保数据存在。
  • 周/月视图在应用层聚合，计算平均分。
  • avg_score 是所有数据点的总平均值，用于 markLine。

💡 重点理解 这个接口的核心价值是 “基于数据库的按需聚合”——数据来源是已持久化的报告，缺失时自动补全，保证了趋势分析的一致性和可靠性。

for (let d = new Date(weekStart); d <= weekEnd; d.setDate(d.getDate() + 1)) { const dateStr = d.toISOString().split('T')[0]; weekSum += await getOrCreateDailyScore(userId, deviceId, dateStr); weekCount++; } const avg = parseFloat((weekSum / weekCount).toFixed(1)); labels.push(`第${getWeekNumber(weekStart)}周`); scores.push(avg); totalSum += avg; totalCount++; } } else if (period === 'month') { const startOfMonth = new Date(baseDate.getFullYear(), baseDate.getMonth(), 1); for (let m = 5; m >= 0; m--) { const monthStart = new Date(startOfMonth); monthStart.setMonth(monthStart.getMonth() - m); const monthEnd = new Date(monthStart.getFullYear(), monthStart.getMonth() + 1, 0); let monthSum = 0, monthCount = 0; for (let d = new Date(monthStart); d <= monthEnd; d.setDate(d.getDate() + 1)) { const dateStr = d.toISOString().split('T')[0]; monthSum += await getOrCreateDailyScore(userId, deviceId, dateStr); monthCount++; } const avg = parseFloat((monthSum / monthCount).toFixed(1)); labels.push(`${monthStart.getFullYear()}-${String(monthStart.getMonth()+1).padStart(2,'0')}`); scores.push(avg); totalSum += avg; totalCount++; } } const avgScore = parseFloat((totalSum / totalCount).toFixed(1)); res.json({ code:0, message:'success', data: { period, labels, scores, avg_score: avgScore } }); });

❗ 七、常见问题及解决方案

❌ 问题1：汇总接口返回 "Cannot GET /api/sleep/summary"

原因：路由未正确添加，或 app.js 中路由顺序问题。

解决方案：检查 app.js 中是否包含 app.get('/api/sleep/summary', ...)，确保在 app.use(express.json()) 之后定义。

❌ 问题2：日视图正常，但周/月视图数据不对

原因：周/月聚合逻辑有误，或日期计算错误。

解决方案：检查周/月视图的日期范围计算，确保正确获取每周/每月的起始和结束日期。

❌ 问题3：切换视图后图表不刷新

原因：updateSummaryChart 未在数据加载后调用，或图表实例未正确保存。

解决方案：确保在 loadSummary 的 success 回调中调用 this.updateSummaryChart()。

❌ 问题4：markLine 不显示

原因：data.avg_score 未正确传递，或 markLine 配置错误。

解决方案：检查后端返回的 avg_score 字段是否存在，确保 markLine 的 yAxis 值正确。

❌ 问题5：周视图的周数显示为负数或错误

原因：getWeekNumber 函数实现有误。

解决方案：使用标准周数计算逻辑，确保返回正确的周数。

❌ 问题6：数据库中没有生成缺失日期的报告

原因：getOrCreateDailyScore 中的 INSERT 未执行，或 saveDb() 未调用。

解决方案：检查 getOrCreateDailyScore 的异常处理，确保插入成功并调用 saveDb()。查看数据库确认记录是否生成。
