3.1 user_settings 表设计回顾（约3分钟）

💡 教师口述
后端使用的 `user_settings` 表结构如下（对齐第1节 ER 图）：
字段名类型约束说明
user_idINTEGERUNIQUE, FK用户ID，一对一
bed_timeTEXT就寝时间 (HH:mm)
wake_timeTEXT起床时间 (HH:mm)
sunrise_duration_minutesINTEGERDEFAULT 10日出模拟时长（分钟）
💡 关键设计 `user_id` 是 UNIQUE 的，所以每个用户只有一条记录。后端使用 `INSERT OR REPLACE` 实现“有则更新，无则插入”。

3.2 小程序组件：picker 和 slider（约4分钟）

💡 教师口述
  • <picker mode="time">：系统原生时间选择器。用户滑动选择小时和分钟，通过 `bindchange` 事件返回 `e.detail.value`（格式 "HH:mm"）。
  • <slider>：滑动输入条。通过 `bindchange` 事件返回 `e.detail.value`（数字）。我们用来控制日出模拟时长（5-30分钟）。
今天我们会把这两个组件和 `wx.request` 结合起来，完成数据的双向绑定。

A. 后端代码走读：理解已有接口

✅ A-任务1：走读 app.js 中的设置接口 ⏱️ 10分钟
📚 前置知识：后端已实现的功能
打开 `backend/app.js`，找到“用户作息设置”部分（约在第 390-480 行）。
你需要理解两个接口：
  • GET /api/setting/plan：查询当前用户的设置，若未设置返回默认值（不写入数据库）。
  • PUT /api/setting/plan：接收 `bed_time`、`wake_time`、`sunrise_duration_minutes`，使用 `INSERT OR REPLACE` 写入数据库。
🔍 重点代码解析（PUT 接口的合并策略）
// 如果用户只传了 bed_time，则 wake_time 等字段需保留原值或默认值 const existing = dbGetOne(db, 'SELECT ... FROM user_settings WHERE user_id = ?', [userId]); const merged = { bed_time: bed_time !== undefined ? bed_time : (existing ? existing.bed_time : DEFAULT_SETTINGS.bed_time), wake_time: wake_time !== undefined ? wake_time : (existing ? existing.wake_time : DEFAULT_SETTINGS.wake_time), sunrise_duration_minutes: sunrise_duration_minutes !== undefined ? parseInt(...) : (existing ? existing... : DEFAULT...), };
  • 部分更新：前端可以只传一个字段，后端自动合并其他字段。
  • 默认值：`DEFAULT_SETTINGS` 定义了 `bed_time: '23:00'`, `wake_time: '07:00'`, `sunrise_duration_minutes: 10`。
  • INSERT OR REPLACE：利用 UNIQUE 约束，存在则替换，不存在则插入。

💡 重点理解 这种设计让前端可以灵活地只修改用户想改的项，而不必每次都传全部字段。

  2. 代码走读：核心逻辑详解

    1. 数据加载（loadSettings）

    loadSettings() { const token = getApp().getToken(); if (!token) return wx.redirectTo({ url: '/pages/login/login' }); this.setData({ loading: true }); wx.request({ url: 'http://127.0.0.1:3000/api/setting/plan', method: 'GET', header: { 'Authorization': `Bearer ${token}` }, success: (res) => { if (res.data.code === 0) { this.setData({ bedTime: res.data.data.bed_time, wakeTime: res.data.data.wake_time, sunriseDuration: res.data.data.sunrise_duration_minutes, }); } }, complete: () => this.setData({ loading: false }) }); }
    • 接口返回的字段是 `bed_time`（下划线），前端使用驼峰 `bedTime`，注意映射。
    • 后端如果未设置，会返回默认值（23:00, 07:00, 10），所以 `loadSettings` 总能成功回显。

    2. 保存数据（saveSettings）

    saveSettings() { const token = getApp().getToken(); const { bedTime, wakeTime, sunriseDuration } = this.data; this.setData({ saving: true }); wx.request({ url: 'http://127.0.0.1:3000/api/setting/plan', method: 'PUT', header: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` }, data: { bed_time: bedTime, wake_time: wakeTime, sunrise_duration_minutes: sunriseDuration }, success: (res) => { if (res.data.code === 0) { wx.showToast({ title: '保存成功', icon: 'success' }); } else { wx.showToast({ title: res.data.message, icon: 'none' }); } }, complete: () => this.setData({ saving: false }) }); }
    • 注意请求体字段名要和后端一致：`bed_time`、`wake_time`、`sunrise_duration_minutes`。
    • 因为后端支持部分更新，我们传全量字段也没问题。

❗ 七、常见问题及解决方案

❌ 问题1：点击首页按钮跳转报错 "页面不存在"

原因：`app.json` 中未正确添加 `pages/settings/settings`，或路径拼写错误。

解决方案：检查 `app.json` 的 `pages` 数组，确保路径正确，保存后重新编译。

❌ 问题2：进入设置页后数据不显示（空白）

原因：`loadSettings` 请求失败，或 `data` 绑定错误。

解决方案：在 `success` 回调中 `console.log(res)` 查看返回数据；检查字段名映射（后端是 `bed_time`，前端用 `bedTime`）。

❌ 问题3：点击保存后提示 "保存失败"

原因：请求体字段名与后端不匹配（如使用了驼峰 `bedTime`）。

解决方案：检查 `data` 对象，确保使用 `bed_time`、`wake_time`、`sunrise_duration_minutes`。

❌ 问题4：保存成功但重新进入页面数据未更新

原因：`loadSettings` 仍在 `onLoad` 中只执行一次，但 `onShow` 未触发刷新。

解决方案：将 `this.loadSettings()` 也放在 `onShow` 中，或保存成功后直接更新本地 `data`（但推荐从服务器重新拉取以确保一致性）。
