3.1 doctor_authorizations 表设计回顾（约4分钟）

💡 教师口述
根据第1节设计的 ER 图，doctor_authorizations 表结构如下（已在第2节创建）：
字段名类型约束说明
idINTEGERPRIMARY KEY主键，自增
patient_idINTEGERNOT NULL, FK→users患者ID（授权发起方）
doctor_idINTEGERNOT NULL, FK→users医生ID（授权接收方）
statusTEXTDEFAULT 'pending'状态：pending / active / revoked / expired
expire_dateTEXT过期日期（YYYY-MM-DD）
doctor_noteTEXT医生备注
requested_atTEXTDEFAULT (datetime('now'))请求时间
responded_atTEXT响应时间
💡 关键设计
  • 状态机：pending（待确认）→ active（已授权）→ revoked（已撤销）→ expired（已过期）
  • 唯一约束：同一患者 + 同一医生 + 非撤销状态，只能有一条有效授权
  • 过期时间：默认 30 天有效期，到期后状态自动变为 expired

3.2 授权业务规则（约4分钟）

💡 教师口述
  • 授权流程：患者输入医生手机号 → 系统查找医生 → 创建授权记录（status='pending'）→ 医生确认后变为 active
  • 撤销授权：患者可以随时撤销授权（status → 'revoked'）
  • 过期处理：授权默认 30 天有效，后端查询时自动过滤已过期的记录
  • 防止重复：同一医生如果已有 pending 或 active 授权，不允许重复授权
⚠️ 本节简化：今天我们只实现“用户侧”的授权管理——发起授权、撤销授权、查看列表。医生确认授权（pending → active）将在第10节（医生端）实现。

3.3 涉及的 SQL 操作（约4分钟）

💡 教师口述
今天会用到四种 SQL 操作：
  • INSERT：创建授权记录
  • SELECT ... JOIN：查询已授权医生列表（关联 users 表获取医生姓名）
  • UPDATE：撤销授权（更新 status）
  • SELECT：检查是否已存在授权（防止重复）
特别注意：我们使用 JOIN 查询，从 doctor_authorizations 关联 users 表，获取医生的手机号和昵称。

🔍 代码走读：核心逻辑详解

1. 授权接口（POST /api/doctor/grant）

app.post('/api/doctor/grant', authenticateToken, async (req, res) => { const patientId = req.user.id; const { doctor_phone } = req.body; // 1. 查找医生 const db = await getDb(); const doctor = dbGetOne(db, 'SELECT id, nickname FROM users WHERE phone = ? AND role = "doctor"', [doctor_phone]); if (!doctor) { return res.json({ code: 1001, message: '该手机号不是医生', data: null }); } // 2. 检查是否已授权（pending 或 active） const existing = dbGetOne(db, 'SELECT id FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ? AND status IN ("pending", "active")', [patientId, doctor.id]); if (existing) { return res.json({ code: 1001, message: '已授权该医生', data: null }); } // 3. 计算过期时间（30天后） const expireDate = new Date(); expireDate.setDate(expireDate.getDate() + 30); const expireStr = expireDate.toISOString().split('T')[0]; // 4. 插入授权记录 const now = new Date().toISOString(); db.run('INSERT INTO doctor_authorizations (patient_id, doctor_id, status, expire_date, requested_at, created_at, updated_at) VALUES (?, ?, "pending", ?, ?, ?, ?)', [patientId, doctor.id, expireStr, now, now, now]); saveDb(); // 5. 返回结果 const newAuth = dbGetOne(db, 'SELECT * FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ?', [patientId, doctor.id]); return res.json({ code: 0, message: '授权成功', data: newAuth }); });
  • 角色校验：只有 role='doctor' 的用户才能被授权。
  • 防重复：使用 status IN ("pending", "active") 检查是否已有有效授权。
  • 过期时间：当前日期 + 30 天，格式为 YYYY-MM-DD。
  • 状态：初始为 pending（待医生确认）。

2. 撤销接口（DELETE /api/doctor/revoke）

app.delete('/api/doctor/revoke', authenticateToken, async (req, res) => { const patientId = req.user.id; const { doctor_id } = req.body; const db = await getDb(); // 查找有效授权 const auth = dbGetOne(db, 'SELECT id FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ? AND status IN ("pending", "active")', [patientId, doctor_id]); if (!auth) { return res.json({ code: 1001, message: '未找到该授权', data: null }); } // 更新状态为 revoked const now = new Date().toISOString(); db.run('UPDATE doctor_authorizations SET status = "revoked", updated_at = ? WHERE id = ?', [now, auth.id]); saveDb(); return res.json({ code: 0, message: '撤销成功', data: null }); });
  • 软删除：不真正删除记录，而是将 status 改为 revoked，保留审计日志。
  • 权限校验：只允许患者撤销自己的授权（通过 patient_id 匹配）。

3. 列表接口（GET /api/doctor/granted）

app.get('/api/doctor/granted', authenticateToken, async (req, res) => { const patientId = req.user.id; const db = await getDb(); // JOIN 查询获取医生信息 + 过滤过期 const sql = ` SELECT a.id, a.doctor_id, a.status, a.expire_date, a.requested_at, u.nickname as doctor_name, u.phone as doctor_phone FROM doctor_authorizations a JOIN users u ON a.doctor_id = u.id WHERE a.patient_id = ? AND a.status IN ('pending', 'active') AND a.expire_date >= date('now') ORDER BY a.requested_at DESC `; const list = dbGetAll(db, sql, [patientId]); return res.json({ code: 0, message: 'success', data: list }); });
  • JOIN 查询：同时获取授权信息和医生信息，避免两次查询。
  • 过滤过期：使用 expire_date >= date('now') 过滤已过期的授权。
  • 排序：最新的授权排在前面。

// ========== 医生授权 API ========== // POST /api/doctor/grant —— 授权医生 app.post('/api/doctor/grant', authenticateToken, async (req, res) => { const patientId = req.user.id; const { doctor_phone } = req.body; if (!doctor_phone) { return res.json({ code: 1001, message: '请输入医生手机号', data: null }); } const db = await getDb(); // 查找医生 const doctor = dbGetOne(db, 'SELECT id, nickname FROM users WHERE phone = ? AND role = "doctor"', [doctor_phone]); if (!doctor) { return res.json({ code: 1001, message: '该手机号不是医生', data: null }); } // 检查是否已授权 const existing = dbGetOne(db, 'SELECT id FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ? AND status IN ("pending", "active")', [patientId, doctor.id]); if (existing) { return res.json({ code: 1001, message: '已授权该医生', data: null }); } // 计算过期时间（30天） const expireDate = new Date(); expireDate.setDate(expireDate.getDate() + 30); const expireStr = expireDate.toISOString().split('T')[0]; const now = new Date().toISOString(); db.run('INSERT INTO doctor_authorizations (patient_id, doctor_id, status, expire_date, requested_at, created_at, updated_at) VALUES (?, ?, "pending", ?, ?, ?, ?)', [patientId, doctor.id, expireStr, now, now, now]); saveDb(); const newAuth = dbGetOne(db, 'SELECT * FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ?', [patientId, doctor.id]); return res.json({ code: 0, message: '授权成功', data: newAuth }); }); // DELETE /api/doctor/revoke —— 撤销授权 app.delete('/api/doctor/revoke', authenticateToken, async (req, res) => { const patientId = req.user.id; const { doctor_id } = req.body; if (!doctor_id) { return res.json({ code: 1001, message: '请指定医生', data: null }); } const db = await getDb(); const auth = dbGetOne(db, 'SELECT id FROM doctor_authorizations WHERE patient_id = ? AND doctor_id = ? AND status IN ("pending", "active")', [patientId, doctor_id]); if (!auth) { return res.json({ code: 1001, message: '未找到该授权', data: null }); } const now = new Date().toISOString(); db.run('UPDATE doctor_authorizations SET status = "revoked", updated_at = ? WHERE id = ?', [now, auth.id]); saveDb(); return res.json({ code: 0, message: '撤销成功', data: null }); }); // GET /api/doctor/granted —— 已授权医生列表 app.get('/api/doctor/granted', authenticateToken, async (req, res) => { const patientId = req.user.id; const db = await getDb(); const sql = ` SELECT a.id, a.doctor_id, a.status, a.expire_date, a.requested_at, u.nickname as doctor_name, u.phone as doctor_phone FROM doctor_authorizations a JOIN users u ON a.doctor_id = u.id WHERE a.patient_id = ? AND a.status IN ('pending', 'active') AND a.expire_date >= date('now') ORDER BY a.requested_at DESC `; const list = dbGetAll(db, sql, [patientId]); return res.json({ code: 0, message: 'success', data: list }); });

❗ 七、常见问题及解决方案

❌ 问题1：授权时提示“该手机号不是医生”

原因：目标用户的 role 字段不是 'doctor'。

解决方案：在 SQLite 中手动将该用户的 role 改为 'doctor'： UPDATE users SET role = 'doctor' WHERE phone = '13800138001';

❌ 问题2：授权时提示“已授权该医生”

原因：该医生已有 pending 或 active 状态的授权记录。

解决方案：这是正常的业务逻辑，防止重复授权。如果确实需要重新授权，先撤销再添加。

❌ 问题3：列表不显示已授权的医生

原因：可能被过期过滤掉了（expire_date >= date('now')），或 status 不是 pending/active。

解决方案：检查数据库中的 expire_date 和 status 字段值是否正确。

❌ 问题4：撤销后列表未更新

原因：loadList() 未被调用，或请求失败。

解决方案：在撤销成功的回调中确保调用了 this.loadList()，并检查网络请求是否成功。

❌ 问题5：添加授权时卡住（loading 一直转）

原因：请求未返回，可能服务器未启动或接口地址错误。

解决方案：检查服务器是否运行，确认 BASE_URL 是否正确（http://127.0.0.1:3000）。

❌ 问题6：doctor_authorizations 表不存在

原因：第2节数据库初始化时未正确创建该表。

解决方案：使用 A-任务1 中的备用提示词，让 Claude 在 schema.js 中添加建表语句，然后重启服务器。
